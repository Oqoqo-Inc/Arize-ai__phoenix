Sessions enable tracking and organizing related traces across multi-turn conversations with your AI application. When building conversational AI, maintaining context between interactions is critical - Sessions make this possible from an observability perspective.

With Sessions in Phoenix, you can:

* Track the entire history of a conversation in a single thread

* View conversations in a chatbot-like UI showing inputs and outputs of each turn

* Search through sessions to find specific interactions

* Track token usage and latency per conversation

This feature is particularly valuable for applications where context builds over time, like chatbots, virtual assistants, or any other multi-turn interaction. By tagging spans with a consistent session ID, you create a connected view that reveals how your application performs across an entire user journey.

## Session Management

Sessions are visible in the UI, but you can also manage them programmatically when you need to inspect, annotate, or clean up conversation history.

* Use `client.sessions.get_session_turns()` or `getSessionTurns()` to reconstruct ordered turns from a session's root spans.
* Use `client.sessions.delete()` or `deleteSession()` to permanently remove a session and its related traces, spans, and annotations.
* Use `client.sessions.bulk_delete()` or `deleteSessions()` to remove multiple sessions in one request.
* Use `client.sessions.log_session_annotations()` or `logSessionAnnotations()` to attach review results, labels, or other metadata to a session.

<Warning>
Deletion is permanent and cascades through traces, spans, and annotations. Bulk deletion also requires all identifiers to use the same identifier type.
</Warning>

<Note>
The session turns API is experimental and derives each turn from the root span's `input.value` and `output.value` attributes. If those attributes are missing, the turn still appears, but its input or output may be empty.
</Note>

## When to use each API

* **View a session** when you only need the conversation timeline in the Phoenix UI.
* **Reconstruct turns** when you need the ordered input/output pairs for downstream processing, export, or debugging.
* **Delete sessions** when you need to remove stale or incorrect conversation history.
* **Log annotations** when you want to capture human or automated feedback on a session without deleting it.
